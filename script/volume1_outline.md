# 第一卷「人族火種」分場大綱 v1.0（劇情聖經 / Single Source of Truth）

> 這份是讓多個子代理「平行擴寫各場對白」時共用的權威大綱。每位擴寫者必須：
> 1. 嚴格遵守自己負責場景的「開場狀態」與「結束狀態」，確保跟前後場景無縫銜接。
> 2. 維持角色語氣（見角色卡），尤其小艾＝毒舌但重情義的軍師、Jack＝理性工程師。
> 3. 把「科學道：觀測→建模→驗證→修正」與「不改大勢、改小勢」的母題自然織進對白，不要說教。
> 4. 輸出格式＝VN step 序列（見最底「step 格式」），只用大綱列出的既有/新素材檔名。

## 卷定位

接續序章結尾（Jack 與小艾活過第一次因果推演，混沌散去、洪荒大地展開，天機顯示「有人要造人」）。

本卷主線：**女媧造人 → Jack 化身「先生」暗中護持人族火種 → 教化文明、累積第一批功德、解鎖手機權限 → 引來聖人目光 → 人族遭遇第一場火種危機 → 小艾完成第一個天道演算法模型，開始質疑天道。**

核心情感：人族脆弱蒙昧，卻有「想活下去、想記住」的韌性。Jack 不稱王不立教，只做一個「先生」。母題＝**文明的火種比個人的功德更重要**。

卷末鉤子：接第二卷（巫妖大劫）。

## 七場分場 Beat（每場標明：開場狀態 / 內容重點 / 結束狀態 / 用到的素材 / 是否分支）

### 場景 c1_creation —「泥中生靈」
- **開場狀態**：承序章。Jack 與小艾立於洪荒大地，天機因果線匯聚於遠方。
- **內容**：兩人遠觀女媧造人。女媧以黃土仿自身造人，吹靈氣賦生。小艾以科學道觀測：人族個體脆弱、壽短、無修為，但「文明存續機率」這條曲線有奇異的韌性——她第一次對一個「物種」感到數據上的意外。Jack 看著泥胚睜眼、學步、發出第一聲呼喊，動容。女媧立繪短暫現身（遠景／神性），不與 Jack 正面交談，造完人即隱去。
- **結束狀態**：人族初生於大地，蒙昧脆弱。Jack 決定靠近觀察。
- **素材**：bg=bg_creation；sprite=jack/xiaoai_neutral/xiaoai_serious/nuwa；無手機事件或僅輕觀測。
- **分支**：無。next→c2_firstcontact

### 場景 c2_firstcontact —「該不該出手」
- **開場狀態**：Jack 潛近一個人族小部落。部落正面臨困境（寒夜將至、不會保存火種，火堆一滅就陷入黑暗與野獸威脅；小女孩阿禾的弟弟病弱）。
- **內容**：因果雷達顯示——若不介入，這個部落今冬大概率覆滅，但這只是「小勢」，不影響洪荒大勢。Jack 面臨穿越以來第一個道德工程抉擇。小艾提醒「不改大勢、改小勢」是最穩打法，但介入會留下因果痕跡、增加被推演風險。**玩家選擇**：(A) 謹慎——只留下「取火保存」的線索悄悄幫一把，不露面（低風險）；(B) 直接現身教他們（高風險高回報，更快建立信任）。兩條路在 c3 匯流，但會被小艾記成不同的「行事風格樣本」。
- **結束狀態**：Jack 以某種方式幫部落渡過第一夜，與阿禾、石爺初次產生連結。
- **素材**：bg=bg_tribe_night；sprite=jack/xiaoai_serious/xiaoai_neutral/ahe/elder；phone=radar_alert（顯示部落存續機率）。
- **分支**：有（A 謹慎 / B 現身），兩路 next 都→c3_teacher

### 場景 c3_teacher —「做先生」
- **開場狀態**：Jack 已被部落接納（依 c2 選擇，或為神秘恩人、或為明面上的「先生」）。
- **內容**：蒙太奇式推進 Jack 不稱王、不立教，只教實用之學：鑽木與火種保存、結繩／刻符記事、辨識草藥與煮水防疫、簡單農耕與儲糧、壕溝與柵欄防獸。阿禾成為他第一個、最聰慧的學生（學會「記號」並反過來教其他孩子）。石爺從戒備到心折。小艾在旁吐槽兼建模：每一項技術擴散，都讓她的「文明存續曲線」往上跳。穿插溫暖日常與小艾少見的、藏在毒舌下的柔軟。
- **結束狀態**：部落興旺，文明火種點亮。一股微光開始在 Jack 身上凝聚（功德前兆）。
- **素材**：bg=bg_tribe_day / bg_tribe_night；sprite=jack/xiaoai_neutral/xiaoai_smug/ahe/elder；phone 可顯示 ui_codex（文明資料庫建立中）。
- **分支**：無。next→c4_merit

### 場景 c4_merit —「功德與權限」
- **開場狀態**：人族文明擴散，氣運匯聚。
- **內容**：澄澈天穹灑下功德金光，落於 Jack（教化之功）。手機「功德帳本」首次啟動並結算第一筆穩定功德；以功德為憑，合道鎖鬆開一階，解鎖「局部補丁」（修小範圍法則漏洞）與「功德帳本」權限。小艾把功德建模為「天道對世界穩定性貢獻的正回饋」，興奮於「玄學現象第一次給出可量化的回饋訊號」——這是科學道的重要一步。Jack 冷靜：權限是工具，不是目的；他更在意人族能不能自己活下去。
- **結束狀態**：手機升一階，兩人首次嚐到「修補天道換取合法權限」的工程式升權甜頭。
- **素材**：bg=bg_merit_light；sprite=jack/xiaoai_smug/xiaoai_serious；phone=merit_gain（顯示 ui_merit）、patch_unlock（顯示 ui_patch）。
- **分支**：無。next→c5_gaze

### 場景 c5_gaze —「聖人的目光」
- **開場狀態**：人族驟興，引動天機。
- **內容**：人族的異常興盛在天機上劃出一道不該如此快的軌跡，一道高位格目光順因果落下（power）。對方察覺「人族背後有外力＝Jack 的痕跡」。緊張對峙：不是要殺，是要「看清」。小艾調滿因果雷達周旋、適時遮蔽，同時反向採樣，補完她在序章採到的那筆樣本——確認對方是一位**關注／護持人族的高位存在**（暗示與女媧一脈相關，不點明）。對方留下一句意味深長的話（大意：「教而不取，倒是個異數。我記住你了。」）便收回目光。危機解除，但 Jack 正式進入「被頂層注意到」的名單。
- **結束狀態**：Jack 暴露於高層視野但未被敵視，反得到一種微妙的默許。
- **素材**：bg=bg_starfield_dao / bg_tribe_day；sprite=jack/xiaoai_serious/xiaoai_shock/power；phone=radar_alert、cloak_on。
- **分支**：無（保留一次小型手機互動：玩家點「遮蔽」化解）。next→c6_crisis

### 場景 c6_crisis —「火種危機」
- **開場狀態**：一場針對部落的災劫降臨（天災／妖獸潮，作為日後巫妖大劫的遠雷）。
- **內容**：災劫屬「小勢中的劇變」，Jack 無法逆轉整場災（大勢不可違），只能減災。時間與資源有限，必須取捨。**玩家選擇（情感重量最大的一次）**：(A) **救最多的人**——集中力量護住族人撤離，但承載文明的器物／記事符／種糧可能燒毀；(B) **保住文明火種**——拚死搶出火種、記事、種糧與知識載體，讓文明能在別處重生，但救援人力被分散、傷亡可能更高。小艾不替他做決定，只把兩條路的「代價模型」冷靜攤開——這正是科學道：不逃避取捨，只求更低代價、更少犧牲的最優解。阿禾在此役中展現成長（用學到的知識救人／護火種）。災後有傷痛，但火種與大部分人留下了。
- **結束狀態**：部落重創但存續，文明火種延續。Jack 與小艾沉默良久。
- **素材**：bg=bg_calamity；sprite=jack/xiaoai_serious/xiaoai_neutral/ahe/elder；phone=radar_alert、patch_unlock（局部補丁減災）。
- **分支**：有（A 救人 / B 保火種），兩路 next 都→c7_finale

### 場景 c7_finale —「第一行天道審計」
- **開場狀態**：災後，部落於廢墟上重建，篝火重新燃起。
- **內容**：小艾基於這場災劫的全程數據，完成她第一個完整的「天道演算法」局部模型——並指出一個冰冷的發現：這場災劫的清算邏輯**粗暴、低效、可避免**，天道在「維持大勢穩定」時，把眾生當成可損耗的變數。她第一次明確說出對天道的質疑：「天數如此」不是不可挑戰的真理，而是一個**沒被驗證過的、可能存在更優解的演算法**。Jack 望著重燃的篝火與長大的阿禾，立下本卷的志：他要的不是成聖、不是逆天，是「讓天道也必須被審計」。卷末，手機因果雷達深處浮現遠方兩股龐大氣運對撞的前兆（巫妖）——下一場，是真正的大劫。
- **結束狀態**：人族火種立穩，小艾踏出證科學道的第一步，巫妖大劫的陰影浮現。
- **素材**：bg=bg_tribe_night / bg_merit_light；sprite=jack/xiaoai_serious/xiaoai_smug/ahe；phone=radar_alert（巫妖氣運前兆）。
- **分支**：無。next=null（卷末）。卷末旁白點題＋「第二卷『巫妖大劫』製作中」。

## step 格式（嚴格遵守，與序章 prologue.json 完全一致）

每個場景輸出為一個物件：
```
{
  "id": "場景id",
  "steps": [
    { "bg": "背景檔名" },
    { "sprite": "立繪檔名" },
    { "item": "道具檔名" },
    { "who": "角色key", "text": "對白或旁白" },
    { "phone": "事件名", "label": "手機面板顯示文字" },
    { "choice": [ { "text": "選項文字", "goto": "目標場景id" }, ... ] }
  ],
  "next": "下一場景id 或 null"
}
```
- 一個 step 可同時含 sprite + who + text（換立繪並說話）。
- who 只能用：narration / jack / xiaoai / tiandao / power / nuwa / ahe / elder。
- 旁白用 who:"narration"。
- phone 事件名可用：boot / radar_alert / cloak_on / merit_gain / patch_unlock。（新事件 merit_gain、patch_unlock 由引擎擴充支援；label 寫面板提示文字。）
- 立繪檔名只能用角色卡列出的；背景／道具只能用新素材清單與既有素材的檔名。
- 對白要飽滿、有文學性與畫面感，但保持各角色語氣。每場對白量約為序章單場的 1.5–2 倍。
